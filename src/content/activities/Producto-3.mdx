    <Title title="¿Qué es un algoritmo de búsqueda?" size="2xl" />
    
    <p>
      Nos permite localizar elementos con ciertas propiedades dentro de una estructura
      de datos; por ejemplo, ubicar el registro correspondiente a cierta persona en una
      base de datos, o el mejor movimiento de una partida de ajedrez.
    </p>

    <Title title="¿Qué es un algoritmo de ordenamiento?" size="2xl" />

    <p>
      Conjunto de instrucciones que toman una estructura de datos y organizan los elementos en un orden, (numerico, alfabético, etc).
    </p>
  </section>

  <section class="content flex flex-col gap-2" data-section-title="Búsqueda lineal" id="lineal-search">
    <Title title="Búsqueda lineal" size="4xl" center={true}/>

    <p>Caracterizado por recorrer una estructura de datos, elemento por elemento, de inicio a fin comparando cada valor existente con el valor que se busca.</p>
    
    <p>
      Cuándo el valor buscado es encontrado, se devuelve la posición de este en la estructura de datos, si esto no se cumple, el usuario verá el siguiente aviso:
      <kbd>El valor no ha sido encontrado</kbd>
    </p>
    
    <p>
      Como desventaja, la complejidad de tiempo en <kbd>O(n)</kbd>, esto nos dice que entre más elementos existan en la estrctura de datos, más tiempo le tomará encontrar el dato.
    </p>

  <section class="content flex flex-col gap-2" data-section-title="Búsqueda binaria" id="binary-search">
    <Title title="Búsqueda binaria" size="4xl" center={true}/>

    <p>
      El algoritmo de búsqueda binario es eficiente en estructuras de datos ordenadas; por ejemplo, arreglos, listas, etc.
    </p>
    
    <p>
      El algoritmo primero ordena la lista de elementos, posteriormente parte la lista en dos mitades, la izquierda y la derecha.

      Posteriormente dentro de cada sub lista hace una busqueda del elemento, si el valor/elemento a búscar se encuentra en la mitad superior o en la mitad inferior (se descarta su contraparte).
    </p>

    <p>
      Esto úlitimo permite que este método de búsqueda sea más eficiente en comparación con el lineal, esto gracias a la reducción en la comparaciones necesarias para encontrar un valor.
    </p>

    <Title title="Ordenamiento recursivo" size="4xl" center={true}/>

    <p>
      Es un algoritmo de ordenamiento recursivo que sigue el principio de "dividir y conquistar". En resumen, divide la lista a ordenar en mitades cada vez más pequeñas hasta que cada mitad solo tenga un elemento. Luego, combina las mitades de forma ordenada, creando sublistas cada vez más grandes, hasta que la lista completa esté ordenada.
    </p>

    <p>
      El proceso de división se realiza de forma recursiva hasta que cada mitad contenga solo un elemento. Luego, se utiliza el proceso de combinación para fusionar cada par de mitades ordenadas en una sola lista ordenada, que se fusiona con la siguiente mitad ordenada hasta que la lista completa esté ordenada.
    </p>
